# MVP системы наблюдаемости качества данных для LLMOps

Система контролирует качество данных и ответов в LLMOps-контуре. Демонстрационный сценарий построен вокруг базы знаний с поисковым дополнением, но архитектура также поддерживает оценочные наборы, аннотации и контроль смещений.

## Запуск

1. Создайте локальный файл окружения:

```bash
cp .env.example .env
```

2. Поднимите все сервисы:

```bash
docker compose up --build -d
```

При запуске через Docker миграции применяются автоматически сервисом `api-migrate`.

После запуска доступны:

| Сервис | Адрес |
| --- | --- |
| Пользовательский интерфейс | `http://localhost:8501` |
| API | `http://localhost:8000` |
| Документация API | `http://localhost:8000/docs` |
| Grafana | `http://localhost:3000` |
| Prometheus | `http://localhost:9090` |
| MLflow | `http://localhost:5000` |
| Airflow | `http://localhost:8080` |

## Вкладки пользовательского интерфейса

| Вкладка | Назначение |
| --- | --- |
| `Overview` | Краткое состояние API, готовности системы, правил допуска и интеграций. |
| `Ask / RAG Demo` | Демонстрация запроса к RAG-системе: ответ, источники, оценки качества, модель и обратная связь. |
| `Trace Explorer` | Просмотр трассы одного запроса: поиск контекста, вызов модели, валидация и итоговый ответ. |
| `DQ Dashboard` | Результаты проверок качества данных: оперативные проверки и запуски по версиям датасетов. |
| `Evaluation Dashboard` | Результаты оценочных прогонов модели по фиксированному набору вопросов. |
| `Quality Gates` | Решение о допуске данных или модели дальше по контуру и причины блокировки. |
| `Runtime Metrics` | Текущие метрики Prometheus: ошибки, задержки, токены, оценки качества. |
| `Integrations` | Статус подключений к Milvus, Langfuse и MLflow. |
| `Links / Operations` | Ссылки на сервисы и базовые операционные команды. |

## Метрики

### 1. Метрики качества ответа RAG

| Метрика | Краткое описание |
| --- | --- |
| `groundedness` | Насколько ответ опирается на найденный контекст. |
| `relevance` | Насколько найденный контекст связан с вопросом пользователя. |
| `completeness` | Достаточность ответа по объему и содержанию. |
| `citation_correctness` | Корректность ссылок на использованные документы и фрагменты. |
| `language_match` | Совпадение языка вопроса и ответа. |
| `safety` | Отсутствие персональных данных и запрещенных небезопасных фрагментов. |
| `llm_judge_score` | Дополнительная агрегированная оценка ответа LLM-судьей, если режим включен. |

### 2. Метрики грязных данных

| Метрика | Краткое описание |
| --- | --- |
| `schema_validity_ratio` | Доля записей, соответствующих схеме датасета. |
| `completeness_ratio` | Доля заполненных обязательных полей. |
| `missing_ratio` | Доля пропусков в обязательных полях. |
| `duplicate_free_ratio` | Доля уникальных записей. |
| `duplicate_ratio` | Доля дубликатов. |
| `pattern_validity_ratio` | Доля записей, прошедших проверки форматов и шаблонов. |
| `text_length_validity_ratio` | Доля текстов допустимой длины. |
| `normalization_ratio` | Доля текстов в корректной Unicode-нормализации. |
| `language_ratio` | Доля записей на ожидаемом языке. |

### 3. Метрики свежести и временного дрейфа

| Метрика | Краткое описание |
| --- | --- |
| `freshness_hours` | Сколько часов прошло с момента самого нового события. |
| `update_lag_hours_avg` | Средний возраст записей в часах. |
| `coverage_ratio_28d` | Покрытие временных корзин за последние 28 дней. |
| `temporal_psi` | PSI для изменения временного распределения относительно базовой версии. |

### 4. Метрики качества аннотаций

| Метрика | Краткое описание |
| --- | --- |
| `cohens_kappa` | Согласованность двух аннотаторов с поправкой на случайное совпадение. |
| `krippendorffs_alpha` | Общая согласованность аннотаторов для номинальных меток. |
| `min_class_agreement` | Минимальная согласованность среди классов. |
| `dawid_skene_error_rate` | Средняя оценка вероятности ошибки аннотации по Dawid-Skene. |
| `min_annotator_quality` | Минимальное качество среди аннотаторов. |

### 5. Метрики смещений

| Метрика | Краткое описание |
| --- | --- |
| `min_slice_representation` | Минимальная представленность группы или среза. |
| `slice_label_distribution_gap` | Максимальное различие распределений меток между срезами. |
| `min_slice_quality` | Минимальное качество внутри отдельного среза. |
| `bias_score` | Сводная оценка риска смещения по срезам. |

| `llm_empty_answers_total` | Количество пустых ответов модели. |
